# 최종 보고서

## 1. 프로그램 주제와 목적

내 프로그램의 주제: 개인 맞춤형 하루 영양소 및 칼로리 충족도 계산기

이 프로그램을 만든 이유: 현대인들은 외식이나 가공식품 섭취가 늘어나면서 영양소를 균형 있게 섭취하기 어려운 경우가 많다. 따라서 사용자가 하루 동안 먹은 음식을 입력하면 총 섭취 칼로리와 탄수화물, 단백질, 지방의 양을 계산하고 목표 칼로리 대비 충족도를 분석하여 건강한 식습관 형성에 도움을 주기 위해 이 프로그램을 제작하였다.
---

## 2. 문제 분해 및 모델링

### 입력 
사용자의 하루 목표 칼로리(kcal)
사용자가 섭취한 음식 이름
음식 섭취량(g)
### 처리
2차원 리스트에서 입력한 음식 정보를 검색한다.
음식의 100g 기준 영양 정보를 이용하여 실제 섭취량에 맞게 칼로리와 영양소를 계산한다.
여러 음식의 영양소를 누적 계산한다.
총 섭취 칼로리를 목표 칼로리와 비교하여 달성률을 계산한다.
조건문을 이용해 영양 상태를 판정한다.
### 출력
총 섭취 칼로리
총 탄수화물, 단백질, 지방 섭취량
목표 칼로리 대비 달성률
영양 상태 판정 결과
---

## 3. 사용한 2차원 리스트 설명

2차원 리스트 이름: food_data

한 행의 의미: 하나의 음식에 대한 영양 정보

각 열의 의미:

| 열 번호 | 의미 | 실제 예시 |
0번 열	음식 이름	닭가슴살
1번 열	100g당 칼로리(kcal)	120
2번 열	100g당 탄수화물(g)	0
3번 열	100g당 단백질(g)	23
4번 열	100g당 지방(g)	1

이 2차원 리스트를 사용한 이유: 음식 이름과 여러 영양 정보를 하나의 데이터로 묶어 저장할 수 있으며, 반복문을 이용해 원하는 음식을 쉽게 검색할 수 있기 때문이다.
---

## 4. 함수 설명

| 함수 이름 | 역할 | 입력값 | 반환값 또는 출력 |
show_menu()	프로그램 안내 출력	없음	메뉴 및 사용법 출력
calculate_nutrition()	음식 영양소 계산	음식 이름, 섭취량	칼로리 및 영양소 리스트 반환
print_report()	최종 결과 출력	총 영양소, 목표 칼로리	결과 보고서 출력

가장 중요한 함수 하나를 골라 자세히 설명하세요.

함수 이름: calculate_nutrition()

설명: 사용자가 입력한 음식 이름을 food_data에서 검색한 후, 해당 음식의 100g 기준 영양 정보를 가져온다. 이후 사용자가 입력한 섭취량에 맞게 비율을 계산하여 실제 섭취한 칼로리, 탄수화물, 단백질, 지방의 양을 구한다. 프로그램의 핵심 계산을 담당하는 가장 중요한 함수이다.

## 5. 조건문과 반복문 활용

### 조건문 사용 부분

음식 이름이 데이터베이스에 존재하는지 검사하였다.
목표 칼로리가 0 이하인지 검사하였다.
섭취량이 0 이하인지 검사하였다.
칼로리 달성률에 따라 에너지 부족, 적정 식사, 에너지 과잉을 판정하였다.


### 반복문 사용 부분

for문을 사용하여 2차원 리스트의 모든 음식을 탐색하였다.
while문을 사용하여 여러 음식을 계속 입력받도록 구현하였다.
잘못된 입력이 들어오면 올바른 값이 입력될 때까지 반복하였다.


### 조건문과 반복문이 함께 작동한 부분

for문으로 food_data를 순회하면서 if문으로 사용자가 입력한 음식 이름과 일치하는지 검사하였다. 또한 while문으로 입력을 반복받는 과정에서 if문을 이용해 종료 여부와 입력값의 유효성을 검사하였다.

## 7. 테스트와 오류 수정

### 테스트 1: 정상 사례

입력:목표 칼로리 2000, 닭가슴살 200g, 현미밥 300g

예상 결과: 총 칼로리와 영양소가 정상적으로 계산되고 최종 보고서가 출력된다.

실제 결과: 예상대로 정상 출력되었다.


### 테스트 2: 예외 또는 경계 사례

입력:존재하지 않는 음식 이름 입력

예상 결과:해당 음식을 찾을 수 없습니다

실제 결과:예상대로 안내 문구가 출력되고 재입력을 받았다.


### 발견한 오류와 수정한 방법
사용자가 음수나 0을 입력하면 계산이 잘못되는 문제가 있었다. 이를 해결하기 위해 조건문을 사용하여 양수만 입력받도록 수정하였다. 또한 숫자가 아닌 문자를 입력할 경우를 대비하여 try-except 구문을 추가하였다.

## 10. 코드인터뷰 준비

1. 내 프로그램의 입력-처리-출력은 무엇인가?

입력은 목표 칼로리와 음식 이름, 섭취량이며, 처리 과정에서는 영양소 계산 및 누적 연산을 수행하고, 출력은 영양 보고서와 충족도 판정 결과이다.

2. 2차원 리스트에서 한 행과 각 열은 무엇을 의미하는가?

한 행은 하나의 음식 정보를 의미하며, 각 열은 음식 이름, 칼로리, 탄수화물, 단백질, 지방 정보를 의미한다.

3. 가장 중요한 함수는 무엇이고 어떤 역할을 하는가?

calculate_nutrition() 함수이며, 음식의 영양소를 실제 섭취량에 맞게 계산하는 핵심 역할을 한다.

4. 조건문과 반복문은 어디에서 사용되었는가?

조건문은 음식 검색, 입력값 검사, 영양 상태 판정에 사용되었고, 반복문은 음식 데이터 탐색과 사용자 입력 반복에 사용되었다.

5. Gemini가 도와준 부분과 내가 직접 수정한 부분은 무엇인가?

Gemini는 구조 설계와 개선 방향을 제안하였고, 최종 코드 구현과 세부 로직 구성은 직접 수정하고 완성하였다.
