# 프로젝트 계획서

## 1. 프로젝트 주제

내가 만들 프로그램의 주제:

개인 맞춤형 하루 영양소 및 칼로리 충족도 계산기

---

## 2. 문제 상황

이 프로그램이 해결하려는 문제를 설명하세요.

예) 사용자가 현재 기분과 사용할 수 있는 시간을 입력하면, 조건에 맞는 활동을 추천해 주고 싶다.

내 설명: 현대인들이 가공식품이나 외식을 할 때 균형 잡힌 영양 섭취를 하기 어렵다는 문제점을 발견했다. 이에 사용자가 먹은 가상 음식을 입력하면 하루 권장량 대비 탄수화물, 단백질, 지방의 성분을 정밀하게 계산해 주어, 영양 과잉이나 결핍을 예방하고자 이 프로그램을 기획했다.


---

## 3. 입력-처리-출력 구조

### 입력

사용자가 입력하거나 프로그램이 사용할 데이터는 무엇인가?

- 사용자의 하루 목표 칼로리 (단위: kcal)   
- 오늘 먹은 가상 음식의 이름과 섭취량 (단위: g) 

### 처리

입력된 데이터를 어떤 규칙이나 알고리즘으로 처리할 것인가?

- 사용자가 입력한 음식을 2차원 리스트 데이터에서 검색 및 매칭   
- 섭취량(g)에 비례하여 실제 섭취한 칼로리, 탄수화물, 단백질, 지방량 계산 (분석·점수 계산형) 

### 출력

프로그램이 최종적으로 무엇을 출력하는가?

- 오늘 총 섭취한 영양성분 요약 결과   
- 목표 대비 영양소 달성률 및 "과잉", "부족", "적정" 등의 영양 판정 문구 출력   
---

## 4. 2차원 리스트 설계

내 프로그램에서 사용할 2차원 리스트의 이름:

예) foods, movies, characters, questions, records

```python
# 가상 식품 영양 성분 데이터 (100g 기준)
food_data = [
    ["닭가슴살", 120, 0, 23, 1],
    ["고구마", 130, 32, 1, 0],
    ["현미밥", 150, 33, 3, 1],
    ["계란후라이", 140, 1, 12, 10]
]

### 한 행의 의미 
가상의 식품 1개에 대한 영양 정보

### 각 열의 의미
0번 열: 식품 이름 (str)   
1번 열: 100g당 칼로리 (int)   
2번 열: 100g당 탄수화물 (g) (int)   
3번 열: 100g당 단백질 (g) (int)   
4번 열: 100g당 지방 (g) (int)

## 5. 함수 설계
1. def show_menu() | 역할: 프로그램의 시작 메뉴와 사용 방법 안내를 화면에 출력. |출력값 (print): 프로그램 타이틀, 가상 식품 데이터 입력 방법, 종료 안내 등의 텍스트 문구
2. def calculate_nutrition(food_name, weight) | 역할: food_data 리스트에서 음식 이름을 찾아, 입력받은 섭취량(weight)에 맞게 칼로리와 영양소를 계산하여 리턴. | 입력값 (매개변수): * food_name (str): 사용자가 입력한 가상 음식 이름: weight (int): 사용자가 입력한 음식 섭취량 (단위: g) | 반환값 (return): 계산된 가상의 [칼로리, 탄수화물, 단백질, 지방] 리스트 또는 튜플
3. def print_report(total_calories, goal_calories) | 역할: 총 섭취량과 목표 칼로리를 비교하여 달성률을 계산하고 영양 상태를 판정하여 출력. |입력값 (매개변수):total_nutrients (list): 누적된 [총 칼로리, 총 탄수화물, 총 단백질, 총 지방] 데이터, goal_calories (int): 사용자가 처음에 설정한 하루 목표 칼로리 | 출력값 (print): * 하루 총 섭취 영양소 요약 보고서칼로리 달성률 (예: 목표 대비 95% 섭취했습니다.), 조건문 판단에 따른 영양 판정 문구 (예: "적정 식사입니다.", "에너지 부족 상태입니다.")  
| 함수 이름 | 역할 | 입력값 | 반환값 또는 출력 |

## 6. 사용할 제어구조

### 조건문을 사용할 부분

예) 사용자의 예산보다 가격이 낮은 항목만 추천한다.

내 계획: 
1. 가상 식품 데이터 매칭 (if)반복문으로 2차원 리스트(food_data)를 검색할 때, 사용자가 입력한 음식 이름과 리스트 0번 열의 이름이 일치하는지 비교할 때 사용합니다.  
2. 입력값 예외 처리 (if ~ else)사용자가 음식 섭취량(g)이나 목표 칼로리를 입력할 때, 0 이하의 숫자나 음수를 입력하면 "올바른 값을 입력하세요"라는 안내를 출력하고 정상적인 계산을 제한하는 방어적 코드를 구현합니다.  
3. 최종 영양 충족도 등급 판정 (if ~ elif ~ else)계산된 '총 섭취 칼로리'와 사용자의 '목표 칼로리'를 비교하여 복합적인 판정을 내립니다.  목표의 80% 미만일 경우: "에너지 부족" 출력   목표의 80% 이상 ~ 120% 이하일 경우: "적정 식사" 출력   목표의 120% 초과일 경우: "에너지 과잉" 경고 출력   


### 반복문을 사용할 부분

예) 2차원 리스트의 모든 항목을 하나씩 검사한다.

내 계획:
1. 2차원 리스트 탐색 (for문)미리 정의해 둔 가상 식품 영양 성분 데이터베이스(2차원 리스트)의 행 개수만큼 반복하며 사용자가 입력한 음식을 찾습니다.  for food in food_data: 구조를 활용하여 리스트 내부의 요소를 하나씩 꺼내어 조건문으로 비교 처리합니다.
2. 다중 음식 입력 및 누적 계산 (while문)사용자가 하루 동안 먹은 음식을 한 가지만 입력하고 끝나는 것이 아니라, 여러 개를 연속해서 입력할 수 있도록 프로그램 전체를 while True: 무한 루프로 제어합니다.사용자가 특정 명령어(예: 종료)를 입력하면 반복문을 탈출(break)하고 최종 결과를 출력하도록 만듭니다.
